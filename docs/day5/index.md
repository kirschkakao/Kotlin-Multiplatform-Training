# Day 5: Frontend Finalization and Scoring

## Overview
Today you'll complete your Carcassonne application by finishing the game flow and integrating scoring.

You will first finalize the game and results screens by adding a player information bar and a proper scoreboard with sorted end results. Then you'll implement follower placement with dedicated board overlays, follower occupancy state, and turn-step transitions in your game logic.

Finally, you'll connect the scoring area model and repository to your gameplay flow so area ownership and points can be processed according to the game rules.


## Before You Start
Make sure you have completed all Day 4 tasks and that your project matches the expected baseline.

!!! warning "Important"
    Whenever you need to import objects from external libraries and have more than one option to choose from, prioritize `androidx.compose` and `material3` imports.

## Tasks Overview

### 1. [Complete the Game and Result Screens](task1.md)
Finalize the game screen with a live player information bar and complete the results screen with a sorted scoreboard and winner indication.


### 2. [Follower Placement](task2.md)
Implement follower placement UI and state handling, including occupancy grids, takeable slots, and follower-placement turn logic.


### 3. [Scoring Areas and Logic](task3.md)
Integrate scoring area structures into the game flow to complete scoring and move the app to a fully playable game.
